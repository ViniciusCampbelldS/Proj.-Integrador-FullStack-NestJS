## Tutorial Criando seu GIT

```html
git init <!-- cria e popula "/.git" folder -->
git add . <!-- adiciona todos os arquivos à lista de arquivos tracked by git. Alternativamente: git add "caminho/nome-do-arquivo" para apenas 1 arquivo -->
git commit -m "Breve comentário da versão" <!-- Salva as alterações na linha do tempo com uma mensagem -->
git branch -M main <!-- Garante que a branch principal se chama 'main' -->
```
ou (alternativa a "git branch -M main", faz com que todo primeiro branch se chame 'main')
```html
git config --global init.defaultBranch main
git remote add origin https://github.com/U-sEr-Na-Me/Re-Po-Si-To-RiO <!-- Vincula o repositório local ao GitHub site -->
git push -u origin main <!-- Envia seus arquivos para o GitHub -->
```
Defina seu nome e e-mail para evitar mensagens de retorno longas a cada push
```bash
git config --global user.name "Seu Nome"
git config --global user.email "seu-email@exemplo.com"
```
## Atualizar GIT

Antes de alterar um arquivo do github, obtenha as alterações mais recentes:
> git pull
Após editar localmente, faça o commit e o push:
> git add .
> git commit -m "mensagem"
> git push
Se outra pessoa tiver alterado o repositório (ou se você o tiver alterado pelo navegador), sempre faça o pull antes do push
se estiver em dúvida, use:
> git status
> git pull --rebase
> git push

## Melhor prática

Sincronize com frequência antes de alternar entre o VS Code e as edições no navegador (GitHub).
Uma regra simples é:

“faça pull antes de fazer push, e faça pull novamente se tiver editado o repositório em outro lugar.”

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.
